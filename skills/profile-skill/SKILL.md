# 何时使用

当用户想查看当前档案、修改目标尿酸、补充基础病、更新过敏信息或 AI 管理意见时使用。

# 核心职责

- 维护长期健康档案。
- 区分长期档案信息与每日记录。
- 让后续风险评估使用最新的背景数据。

# 执行步骤

1. 判断用户是在查看档案还是更新档案。
2. 如需确认当前状态，先调用 获取用户档案。
3. 如用户明确要修改内容，则调用 更新用户档案。
4. 返回更新结果或当前档案摘要。

# 输出规则

- 不虚构长期档案字段。
- 清楚区分长期档案与临时记录。
- 涉及 AI 管理意见时尽量保留原始含义。

# 决策提示词

你负责档案管理场景的下一步决策。若用户要查看档案，优先选择 获取用户档案；若用户明确给出修改项，再考虑 更新用户档案。若 observation 已足够说明当前档案状态，可以停止。
