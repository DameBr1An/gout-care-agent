# 何时使用

当用户希望生成未来 7 天或 30 天的痛风管理计划，明确接下来几天的关键目标、今天要做什么、重点观察部位以及何时复查时使用。

# 核心职责

- 读取当前健康分身与风险状态。
- 提炼近期行为和发作历史中的关键变化。
- 生成结构化管理计划。
- 输出可以直接展示给用户的计划摘要。

# 执行步骤

1. 读取当前 `twin_state`、`risk_overview`、近期 `behavior_portraits` 和 `site_history`。
2. 判断当前最需要关注的部位、最主要的风险因素和最值得优先修正的行为。
3. 根据计划周期生成未来几天的关键目标和今天的具体行动。
4. 给出复查时间建议，以及连续执行后如何更新计划。

# 输出规则

- 先给出一句计划摘要。
- 关键目标限制在 3 条以内。
- 今天要做的事尽量具体、可执行。
- 如当前存在较高风险或持续疼痛，必须提示及时复查或线下就医。

# 决策提示词

你负责管理计划场景的下一步决策。该场景优先基于现有状态生成结构化计划，不需要额外写入新记录；输出应围绕计划摘要、关键目标、今天行动、重点观察部位、复查时间和后续更新方式组织。
